# Lunio Support API

A Node.js realtime support/chat API service for Lunio, a Canadian invoicing SaaS.

## What This Service Is

- A standalone Node.js API service providing realtime support chat functionality
- Uses WebSocket connections via Socket.IO for real-time communication
- Shares Lunio's existing MariaDB database for data access
- Provides health check endpoints for monitoring
- Production-ready foundation for support chat infrastructure

## What This Service Is NOT

- A complete Lunio frontend UI component
- A duplicate authentication system (relies on Lunio's existing auth)
- A separate user/account management system (Lunio remains the source of truth)
- A full chat implementation (only the API/WebSocket foundation)

## Prerequisites

- Node.js 18+
- MariaDB database (shared with Lunio)

## Environment Variables

Copy `.env.example` to `.env` and fill in the required values:

```env
APP_NAME=Lunio Support API
NODE_ENV=development
PORT=3001
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=lunio
DB_USERNAME=your_db_username
DB_PASSWORD=your_db_password
CHAT_TOKEN_SECRET=your_secret_key_for_chat_tokens
LUNIO_APP_URL=https://lunio.ca
CORS_ORIGIN=https://lunio.ca
```

All environment variables are required. The service will not start without them.

## Local Development

1. Install dependencies:
   ```bash
   npm install
   ```

2. Set up your `.env` file (see Environment Variables section)

3. Start the development server:
   ```bash
   npm run dev
   ```

4. The server will start on the port specified in your `.env` file (default: 3001)

## API Endpoints

- `GET /health` - General service health check
- `GET /health/db` - Database connectivity health check
- `GET /dev/test-token` - Development-only endpoint to generate test chat tokens (not available in production)

## Authentication

All Socket.IO connections require a valid signed JWT chat token for authentication. The token must be provided in the connection handshake:

```javascript
const socket = io('http://localhost:3001', {
  auth: {
    token: 'your-jwt-token-here'
  }
});
```

### Token Payload Structure

```json
{
  "sub": 123,           // User ID (required)
  "company_id": 456,    // Company/Account ID (optional)
  "role": "user",       // "user" | "admin" | "support" (required)
  "name": "John Doe",   // Display name (optional)
  "email": "user@example.com", // Email (optional)
  "iat": 1640995200,    // Issued at timestamp (required)
  "exp": 1640998800     // Expiry timestamp (required)
}
```

### Security Note

**Production tokens must be generated by the Lunio PHP application, not by this support-api service.** This ensures Lunio remains the source of truth for authentication and user data. The support-api only verifies tokens and trusts the payload contents when the signature is valid.

## WebSocket

The service accepts authenticated Socket.IO connections. Upon successful connection, it emits a `support:connected` event:

```javascript
socket.on('support:connected', (data) => {
  console.log('Connected:', data);
  // { ok: true, user_id: 123, role: "admin", company_id: 456 }
});
```

## Local Testing

For development, you can generate a test token using the dev endpoint:

```bash
curl http://localhost:3001/dev/test-token
```

This returns a short-lived JWT token with test user data. Use this token in your Socket.IO client for testing:

```javascript
const response = await fetch('http://localhost:3001/dev/test-token');
const { token } = await response.json();

const socket = io('http://localhost:3001', {
  auth: { token }
});
```

The dev endpoint is automatically disabled in production environments.

## Production Deployment

This service is intended to be deployed alongside Lunio using:

- **Nginx** as reverse proxy
- **PM2** for process management

Example PM2 ecosystem file (`ecosystem.config.js`):

```javascript
module.exports = {
  apps: [{
    name: 'lunio-support-api',
    script: 'src/server.js',
    env: {
      NODE_ENV: 'production',
      PORT: 3001,
      // ... other env vars
    }
  }]
};
```

Example Nginx configuration:

```
server {
    listen 80;
    server_name api.lunio.ca;

    location / {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

```
│   └── database.js  # MariaDB connection pool
├── routes/

```

## Security

- Uses Helmet for security headers
- CORS configured for Lunio domain only
- JWT token authentication for WebSocket connections
- No hardcoded secrets
- Environment-based configuration
- Production tokens must be generated by Lunio PHP (not this service)
